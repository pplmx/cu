## Existing CSR SpMV Pattern

```cpp
template<typename T>
void SparseOps<T>::spmv(const SparseMatrixCSR<T>& matrix, const T* x, T* y) {
    int num_rows = matrix.num_rows();
    
    for (int i = 0; i < num_rows; ++i) {
        T sum = T{0};
        for (int idx = matrix.row_offsets()[i]; idx < matrix.row_offsets()[i + 1]; ++idx) {
            int col = matrix.col_indices()[idx];
            sum += matrix.values()[idx] * x[col];
        }
        y[i] = sum;
    }
}
```

## ELL SpMV Algorithm

For ELL format with max_nnz_per_row = M:
```cpp
template<typename T>
void SparseOps<T>::spmv(const SparseMatrixELL<T>& matrix, const T* x, T* y) {
    int num_rows = matrix.num_rows();
    int max_nnz = matrix.max_nnz_per_row();
    
    for (int i = 0; i < num_rows; ++i) {
        T sum = T{0};
        int base = i * max_nnz;
        for (int j = 0; j < max_nnz; ++j) {
            int col = matrix.col_indices()[base + j];
            if (col >= 0) {  // Skip padding
                sum += matrix.values()[base + j] * x[col];
            }
        }
        y[i] = sum;
    }
}
```

Key insight: Iterate through all padded slots but skip negative column indices.

## SELL SpMV Algorithm

For SELL format with slice_height = C:
```cpp
template<typename T>
void SparseOps<T>::spmv(const SparseMatrixSELL<T>& matrix, const T* x, T* y) {
    int num_rows = matrix.num_rows();
    int slice_height = matrix.slice_height();
    int num_slices = (num_rows + slice_height - 1) / slice_height;
    
    for (int s = 0; s < num_slices; ++s) {
        int slice_start_row = s * slice_height;
        int slice_end_row = std::min(slice_start_row + slice_height, num_rows);
        int slice_nnz = (matrix.slice_ptr()[s + 1] - matrix.slice_ptr()[s]) / slice_height;
        
        for (int local_i = 0; local_i < slice_end_row - slice_start_row; ++local_i) {
            int global_i = slice_start_row + local_i;
            T sum = T{0};
            int base = matrix.slice_ptr()[s] + local_i * slice_nnz;
            
            for (int j = 0; j < slice_nnz; ++j) {
                int col = matrix.col_indices()[base + j];
                if (col >= 0) {
                    sum += matrix.values()[base + j] * x[col];
                }
            }
            y[global_i] = sum;
        }
    }
}
```

Key insight: Process slice by slice, compute slice_nnz as padding amount, then standard row-wise SpMV within slice.

## Numerical Accuracy

Both ELL and SELL SpMV must produce bit-exact results as CSR SpMV because:
- All formats store the same values
- All formats use the same column indices
- The multiplication order doesn't affect floating point addition associativity for small additions
- Any row with fewer non-zeros has zero padding that doesn't affect the sum
