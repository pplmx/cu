  <name>Task 1: Create SolverWorkspace class</name>
  <files>include/cuda/sparse/solver_workspace.hpp</files>
  <action>
Create new header file `include/cuda/sparse/solver_workspace.hpp` with:

1. **SolverWorkspace<T>** class that:
   - Accepts matrix size in constructor and pre-allocates all work vectors
   - Provides `r()`, `p()`, `Ap()` (and additional for GMRES: `v()`, `w()`, `cos_sin()`) accessors
   - Includes `resize(int n)` method to handle changing matrix sizes
   - Uses `std::vector<T>` for work vectors (CPU version)
   - Provides `reset()` method to clear vectors without deallocation

2. **SolverDiagnostics** struct that:
   - Stores solver timing information: `setup_time_ms`, `solve_time_ms`, `total_time_ms`
   - Computes `convergence_rate` from residual history (ratio of consecutive residuals)
   - Provides `average_convergence_rate()` returning geometric mean
   - Provides `to_string()` for formatted output

3. **TimedSolverResult<T>** struct that extends SolverResult<T>:
   - Adds `SolverDiagnostics diagnostics` member
   - Inherits all existing SolverResult fields

4. **SolverTimer** helper class:
   - RAII-style timing using cudaEvent_t
   - Stores start/stop events and computes elapsed time

Usage example:
```cpp
SolverWorkspace<double> workspace(1000);  // Pre-allocate for 1000x1000

// First solve
SolverConfig<double> config;
ConjugateGradient<double> solver(config);
workspace.reset();
auto result = solver.solve_with_workspace(A, b, x.data(), workspace);

// Second solve - reuse same workspace (no reallocation)
workspace.reset();
auto result2 = solver.solve_with_workspace(A2, b2, x2.data(), workspace);

// Access diagnostics
std::cout << "Iterations: " << result.iterations << "\n";
std::cout << "Convergence rate: " << result.diagnostics.convergence_rate << "\n";
```

Follow existing code style: namespaces `nova::sparse`, template parameters, const correctness.
