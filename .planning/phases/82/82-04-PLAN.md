Create `docs/krylov_solvers.md` with:

   - What are Krylov subspace methods
   - When to use iterative solvers vs direct solvers
   - Overview: CG (symmetric), GMRES (general), BiCGSTAB (non-symmetric)

2. **Quick Start section**:

   ```cpp
   #include <cuda/sparse/krylov.hpp>

   // Create solver
   SolverConfig<double> config;
   config.relative_tolerance = 1e-8;
   config.max_iterations = 1000;

   ConjugateGradient<double> solver(config);

   // Solve Ax = b
   auto result = solver.solve(A, b.data(), x.data());

   if (result.converged) {
       std::cout << "Converged in " << result.iterations << " iterations\n";
   }
   ```

3. **Solver Configuration section**:
   - relative_tolerance (default: 1e-6)
   - absolute_tolerance (default: 1e-10)
   - max_iterations (default: 1000)
   - verbose (print progress)

4. **Conjugate Gradient (CG)**:
   - Requirements: symmetric positive-definite matrix
   - Example with Laplacian matrix
   - Expected convergence behavior

5. **GMRES**:

   ```cpp
   GMRES<double> solver(config, 50);  // 50 = restart parameter
   auto result = solver.solve(A, b.data(), x.data());
   ```

   - Non-symmetric matrices
   - Restart parameter explanation
   - Memory requirements

6. **BiCGSTAB**:
   - Non-symmetric matrices
   - Often faster than GMRES for some problems
   - Breakdown conditions

7. **Workspace Reuse (KRY-05)**:

   ```cpp
   SolverWorkspace<double> workspace(n);

   // First solve
   solver.solve_with_workspace(A, b, x, workspace);

   // Second solve - reuse workspace
   workspace.reset();
   solver.solve_with_workspace(A2, b2, x2, workspace);
   ```

   - Performance benefits
   - Memory savings

8. **Diagnostics (KRY-06)**:

   ```cpp
   auto result = solver.solve_with_workspace(A, b, x, workspace);

   std::cout << "Iterations: " << result.iterations << "\n";
   std::cout << "Final residual: " << result.residual_norm << "\n";
   std::cout << "Convergence rate: " 
             << result.diagnostics.convergence_rate << "\n";
   ```

   - Available diagnostics
   - Using residual history

9. **Troubleshooting**:
   - "Did not converge" - check matrix is appropriate for solver
   - Slow convergence - try different solver or preconditioner
   - Breakdown - solver-specific issues

10. **API Reference**:
    - SolverConfig
    - SolverResult
    - SolverWorkspace
    - SolverDiagnostics

Use similar structure to sparse_formats.md. Include expected output examples.

   ```cpp
   #include <cuda/sparse/roofline.hpp>

   RooflineAnalyzer analyzer;

   std::cout << "Performance bound: " 
             << RooflineAnalyzer::bound_to_string(metrics.bound) << "\n";
   ```

3. **Understanding Performance Bounds**:
   - **Memory-bound**: Limited by memory bandwidth
   - **Compute-bound**: Limited by arithmetic throughput
   - **Balanced**: Neither limits significantly
   - How to interpret classification

4. **Arithmetic Intensity section**:
   - Formula explanation
   - SpMV arithmetic intensity calculation
   - What affects AI

5. **Device Information**:

   ```cpp
   auto peaks = get_device_peaks();
   // Returns: fp64_peak_gflops, fp32_peak_gflops, fp16_peak_gflops
   //          memory_bandwidth_gbps, compute_capability, multiprocessor_count
   ```

6. **Analyzing Kernels**:

   ```cpp
   RooflineAnalyzer analyzer;
   RooflineMetrics m = analyzer.analyze_kernel(
       "SpMV-CSR",           // kernel name
       2LL * nnz,            // FLOPs
       elapsed_ms,           // measured time
       bytes_accessed,       // memory bytes
       Precision::FP32
   );
   ```

   - Required measurements
   - How to use results

7. **JSON Export (RF-05)**:

   ```cpp
   RooflineAnalysis analysis;
   analysis.add_kernel(metrics);
   analysis.set_device_info(peaks, Precision::FP32, "RTX 3090");

   std::string json = analysis.to_json();
   // Export to file or external visualization
   ```

   - Format explanation
   - Tools for visualization

8. **Performance Tips**:
   - Memory-bound kernels: Optimize memory access patterns
   - Compute-bound kernels: Increase arithmetic intensity
   - Data layout: Consider format changes (CSR vs ELL)
