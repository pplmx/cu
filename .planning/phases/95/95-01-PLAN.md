# Phase 95: Memory Bandwidth Optimization — Plan

## Tasks

### 1. Create RooflineModel (BANDWIDTH-01)

**Files:**
- `include/cuda/performance/bandwidth/roofline_model.h`
- `src/cuda/performance/bandwidth/roofline_model.cpp`

**Implementation:**
- `RooflinePoint` struct with AI, performance, kernel name
- `RooflineModel::add_point()` for trace data
- `RooflineModel::compute()` for AI calculation
- Peak FLOP/s from device properties (FP64/FP32/FP16)
- Peak bandwidth from device properties (HBM)
- JSON export with points, ridge point, rooflines

### 2. Extend BandwidthTracker (BANDWIDTH-02)

**Files:**
- `include/cuda/performance/bandwidth/bandwidth_utilization.h`
- `src/cuda/performance/bandwidth/bandwidth_utilization.cpp`

**Implementation:**
- `BandwidthUtilizationTracker` with historical samples
- Utilization percentage calculation
- Transfer type breakdown (H2D/D2H/D2D)
- Warning thresholds for low utilization
- Integration with existing BandwidthTracker

### 3. Create CacheAnalyzer (BANDWIDTH-03)

**Files:**
- `include/cuda/performance/bandwidth/cache_analyzer.h`
- `src/cuda/performance/bandwidth/cache_analyzer.cpp`

**Implementation:**
- CUpti event-based cache analysis
- L1/L2/texture cache hit rate
- Graceful fallback when events unavailable
- Per-kernel cache metrics

### 4. Create Bandwidth Analysis Module

**Files:**
- `include/cuda/performance/bandwidth/bandwidth_analysis.h`
- `src/cuda/performance/bandwidth/bandwidth_analysis.cpp`

**Implementation:**
- Unified interface for all bandwidth analysis
- Integration with Phase 93 metrics
- Roofline plot data generation

### 5. Write Unit Tests

**File:** `tests/performance/bandwidth_analysis_test.cpp`

**Test cases:**
- RooflineModel AI calculation
- Roofline peak values from device
- Bandwidth utilization tracking
- Cache analyzer graceful degradation
