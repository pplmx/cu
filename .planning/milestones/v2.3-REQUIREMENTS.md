# Requirements: Nova CUDA Library Enhancement — v2.3 Archive

**Milestone:** v2.3 Extended Algorithms
**Shipped:** 2026-04-28
**Core Value:** A reliable, high-performance CUDA compute library that can be trusted in production environments

## v1 Requirements

All requirements shipped in v2.3 milestone.

### Sorting & Searching

- [x] **SORT-01**: User can sort key-value pairs in ascending or descending order using GPU radix sort — Phase 54
- [x] **SORT-02**: User can find the k largest elements in a dataset without performing a full sort — Phase 54
- [x] **SORT-03**: User can perform binary search on sorted arrays using warp shuffle primitives — Phase 54

### Linear Algebra Extras

- [x] **LINALG-01**: User can compute SVD (singular value decomposition) with standard and randomized modes — Phase 55
- [x] **LINALG-02**: User can compute eigenvalues and eigenvectors of symmetric matrices — Phase 55
- [x] **LINALG-03**: User can compute QR, Cholesky, and LDL matrix factorizations — Phase 55

### Numerical Methods

- [x] **NUM-01**: User can run Monte Carlo simulations with variance reduction via antithetic variates — Phase 56
- [x] **NUM-02**: User can compute numerical integrals using trapezoidal and Simpson rules — Phase 56
- [x] **NUM-03**: User can find roots of functions using bisection and Newton-Raphson methods — Phase 56
- [x] **NUM-04**: User can perform interpolation using linear and cubic spline methods — Phase 56

### Signal Processing

- [x] **SIGNAL-01**: User can compute FFT-based convolution for efficient large-kernel filtering — Phase 57
- [x] **SIGNAL-02**: User can compute Haar wavelet transform (forward and inverse) — Phase 57
- [x] **SIGNAL-03**: User can apply FIR (finite impulse response) filters to signals — Phase 57

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Sorting & Searching

- **SORT-10**: User can perform distributed sorting across multiple GPUs (requires NCCL)

### Linear Algebra Extras

- **LINALG-10**: User can compute generalized SVD for rectangular matrices
- **LINALG-11**: User can compute sparse eigensolver (requires cuDSS)

### Signal Processing

- **SIGNAL-10**: User can compute Daubechies wavelet transform
- **SIGNAL-11**: User can implement IIR (infinite impulse response) filters
- **SIGNAL-12**: User can perform continuous wavelet transform

## Out of Scope

Explicitly excluded from v2.3.

| Feature | Reason |
|---------|--------|
| Multi-GPU sorting (NCCL) | Requires multi-node setup; defer to v2.4+ |
| Real-time audio pipeline | Low latency requirements conflict with batch-oriented GPU model |
| Sparse eigensolver (cuDSS) | cuSolverSP deprecated, cuDSS not fully stable |
| Quasi-Monte Carlo (Sobol) | Implementation complexity not justified for v2.3 |
