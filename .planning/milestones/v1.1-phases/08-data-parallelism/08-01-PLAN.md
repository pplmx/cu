## Implementation Details

### 1. Common Infrastructure

#### Stream Per Device Pattern

All collective operations use one CUDA stream per GPU. This is critical: streams are device-local and cannot be shared across GPUs.

```cpp
// include/cuda/distributed/common.h
namespace cuda::distributed {

class MeshStreams {
public:
    MeshStreams();
    ~MeshStreams();

    // Get stream for specific device
    cudaStream_t get_stream(int device);

    // Get event on specific device's stream
    cudaEvent_t get_event(int device);

    // Synchronize all device streams
    void synchronize_all();

    // Barrier: wait for event from src_device on dst_device's stream
    void wait_event(int dst_device, int src_device, cudaEvent_t event);

    int device_count() const { return device_count_; }

private:
    int device_count_;
    std::vector<cudaStream_t> streams_;
    std::vector<cudaEvent_t> events_;
};

}  // namespace cuda::distributed
```

**Key design decision:** Each GPU has its own stream and pre-allocated event. Events are reused across operations to avoid allocation overhead.

#### Data Types

```cpp
namespace cuda::distributed {

enum class ReductionOp {
    Sum,
    Min,
    Max,
    Product
};

// Convert ReductionOp to element-wise CUDA operation
template <typename T>
__device__ T apply_op(T a, T b, ReductionOp op);

}  // namespace cuda::distributed
```

### 2. DistributedReduce — Ring All-Reduce (MGPU-05)

**File:** `include/cuda/distributed/reduce.h`

```cpp
namespace cuda::distributed {

class DistributedReduce {
public:
    // All-reduce across all GPUs in mesh
    // On return, all GPUs have the reduced result in dst
    static void all_reduce(
        const void* send_data,    // Input on each GPU
        void* recv_data,          // Output (same on all GPUs after call)
        size_t count,             // Number of elements
        ReductionOp op,           // Reduction operation
        cudaDataType dtype = cudaDataType::CUDA_R_32F
    );

    // Async variant with explicit streams
    static void all_reduce_async(
        const void* send_data,
        void* recv_data,
        size_t count,
        ReductionOp op,
        cudaStream_t stream,      // Base stream, actual ops use device-specific streams
        cudaDataType dtype = cudaDataType::CUDA_R_32F
    );

    // Check if multi-GPU is needed
    static bool needs_multi_gpu() { return DeviceMesh::instance().device_count() > 1; }
};

}  // namespace cuda::distributed
```

**Implementation:** `src/cuda/distributed/reduce.cu`

Ring all-reduce algorithm (N GPUs, data size M per GPU):

```text
Phase 1: Reduce-scatter (log N steps)
- Step k: Each GPU i sends M/N elements to GPU (i + 2^k) % N
- Each GPU receives and reduces into its buffer
- After log N steps, each GPU has 1/N of the final reduced data

Phase 2: All-gather (log N steps)
- Step k: Each GPU i sends its chunk to GPU (i - 2^k + N) % N
- Each GPU receives and concatenates
- After log N steps, all GPUs have the full reduced result

Total: 2 * log N steps, O(N) bandwidth per GPU
```

**Ring topology:**

- Send to next GPU: `(rank + 1) % n`
- Receive from previous GPU: `(rank - 1 + n) % n`

**Element-wise reduction kernel:**

```cpp
template <typename T, ReductionOp Op>
__global__ void reduce_kernel(T* dst, const T* src, size_t count) {
    size_t idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < count) {
        dst[idx] = apply_op(dst[idx], src[idx], Op);
    }
}
```

**Single-GPU fallback:** Copy send_data to recv_data, skip ring communication.

### 3. DistributedBroadcast — Broadcast (MGPU-06)

**File:** `include/cuda/distributed/broadcast.h`

```cpp
namespace cuda::distributed {

class DistributedBroadcast {
public:
    // Broadcast data from root GPU to all other GPUs
    // All GPUs must call this function with same root and count
    static void broadcast(
        void* data,               // Data on root, receives copy on others
        size_t count,             // Number of elements
        int root = 0,             // Source GPU rank
        cudaDataType dtype = cudaDataType::CUDA_R_32F
    );

    // Async variant
    static void broadcast_async(
        void* data,
        size_t count,
        int root,
        cudaStream_t stream,
        cudaDataType dtype = cudaDataType::CUDA_R_32F
    );
};

}  // namespace cuda::distributed
```

**Implementation:** `src/cuda/distributed/broadcast.cu`

Broadcast algorithm using P2P pipeline:

```text
For each non-root GPU i:
  1. Root copies data to GPU i via PeerCopy
  2. Use event synchronization to ensure ordering

Optimization for adjacent GPUs:
  - If GPU i can directly access root's memory, use direct P2P copy
  - Pipeline: start next copy before waiting for previous to complete
```

**Event-based synchronization:**

```cpp
// Pseudocode for broadcast_async
if (mesh.device_count() == 1 || is_root) {
    // Root or single-GPU: nothing to do
    return;
}

for (int dst = 0; dst < mesh.device_count(); ++dst) {
    if (dst == root) continue;

    PeerCopy copier;
    if (mesh.can_access_peer(root, dst)) {
        copier.copy_async(data, data, count * sizeof(T), dst, root, stream);
    } else {
        // Fallback: use host-mediated copy
        // Copy root data to host, then to dst
        void* tmp = malloc(count * sizeof(T));
        cudaMemcpy(tmp, data, count * sizeof(T), cudaMemcpyDeviceToHost);
        cudaMemcpy(data, tmp, count * sizeof(T), cudaMemcpyHostToDevice);
        free(tmp);
    }
}
```

**Single-GPU fallback:** No-op, data is already in place.

### 4. DistributedAllGather — All-Gather (MGPU-07)

**File:** `include/cuda/distributed/all_gather.h`

```cpp
namespace cuda::distributed {

class DistributedAllGather {
public:
    // All-gather: each GPU contributes data, all GPUs receive concatenated results
    // Each GPU contributes count elements, receives count * n GPUs elements
    static void all_gather(
        const void* send_data,    // Input: count elements from this GPU
        void* recv_data,          // Output: count * n elements (all GPUs' data)
        size_t count,             // Elements per GPU
        cudaDataType dtype = cudaDataType::CUDA_R_32F
    );

    // Async variant
    static void all_gather_async(
        const void* send_data,
        void* recv_data,
        size_t count,
        cudaStream_t stream,
        cudaDataType dtype = cudaDataType::CUDA_R_32F
    );
};

}  // namespace cuda::distributed
```

**Implementation:** `src/cuda/distributed/all_gather.cu`

Ring all-gather algorithm:

```text
For N GPUs with data chunks of size M:
- Each GPU has its chunk at position [rank * M, (rank + 1) * M)
- Goal: each GPU receives all N chunks

Ring communication (N-1 steps):
Step k:
  - Send chunk at position [(rank - k) % N * M, (rank - k + 1) % N * M)
  - Receive into position [(rank + 1) % N * M, ...]
  - Rotate to next position
```

**Alternative: Simple P2P all-gather** (easier to implement, same asymptotic complexity):

```text
For each GPU i != rank:
  - Copy GPU i's data to my buffer at offset i * M
  - Use PeerCopy with event synchronization
```

**Single-GPU fallback:** Copy send_data to recv_data (offset 0).

### 5. MeshBarrier — Barrier Synchronization (MGPU-08)

**File:** `include/cuda/distributed/barrier.h`

```cpp
namespace cuda::distributed {

class MeshBarrier {
public:
    MeshBarrier();
    ~MeshBarrier();

    // Synchronize all GPUs: all must arrive before any proceeds
    void synchronize();

    // Async variant: records event on each device's stream
    // Caller must wait on all returned events
    void synchronize_async(cudaStream_t stream);

    // Synchronize specific subset of GPUs
    void synchronize_devices(const std::vector<int>& devices);

    // Check if barrier is currently engaged (another thread waiting)
    bool is_barriering() const;

private:
    int device_count_;
    std::vector<cudaEvent_t> events_;  // One event per GPU
    std::vector<cudaStream_t> streams_;
};

}  // namespace cuda::distributed
```

**Implementation:** `src/cuda/distributed/barrier.cu`

Barrier synchronization using P2P events:

```text
1. Each GPU records an event on its stream when it reaches barrier
2. Host polls, waiting for all events to complete
3. Once all events recorded, signal all GPUs to proceed

Implementation using cudaEventQuery for polling:
  - Record event E_i on GPU i's stream
  - Poll: for each i, check cudaEventQuery(E_i)
  - When all complete, proceed
```

**Alternative (blocking):**

```text
1. GPU 0 acts as coordinator
2. GPUs 1..N-1 send signal to GPU 0 via peer copy of a flag
3. GPU 0 counts signals and broadcasts "proceed" when all arrive
```

**Single-GPU fallback:** `cudaStreamSynchronize(stream)` on the base stream.

class MeshBarrierTest : public ::testing::Test {
protected:
    void SetUp() override;
};

TEST_F(MeshBarrierTest, BarrierSynchronization) {
    auto& mesh = DeviceMesh::instance();
    int n = mesh.device_count();

    if (n < 2) GTEST_SKIP() << "Requires multiple GPUs";

    MeshBarrier barrier;

    // Create threads that all reach the barrier
    std::vector<std::thread> threads;
    std::atomic<int> arrived(0);
    std::atomic<int> proceeded(0);

    auto barrier_task = [&](int rank) {
        cuda::mesh::ScopedDevice device(rank);
        arrived++;
        barrier.synchronize();
        proceeded++;
    };

    for (int i = 0; i < n; ++i) {
        threads.emplace_back(barrier_task, i);
    }

    for (auto& t : threads) {
        t.join();
    }

    // All should have arrived and proceeded
    EXPECT_EQ(arrived.load(), n);
    EXPECT_EQ(proceeded.load(), n);
}

TEST_F(MeshBarrierTest, NoDeadlock) {
    auto& mesh = DeviceMesh::instance();
    int n = mesh.device_count();

    // This test primarily verifies no deadlock occurs

        EXPECT_NO_THROW(barrier.synchronize());
    }
}

## Implementation Order

1. **Create common infrastructure:** `include/cuda/distributed/common.h`
   - MeshStreams class for per-device stream management
   - ReductionOp enum and apply_op helper

2. **Create `barrier.h` and `barrier.cu`:** MeshBarrier (simplest operation)
   - Event-based synchronization
   - Single-GPU fallback

3. **Create `reduce.h` and `reduce.cu`:** DistributedReduce
   - Ring all-reduce algorithm
   - Reduction kernel
   - Single-GPU fallback

4. **Create `broadcast.h` and `broadcast.cu`:** DistributedBroadcast
   - P2P copy with events
   - Single-GPU fallback

5. **Create `all_gather.h` and `all_gather.cu`:** DistributedAllGather
   - Ring all-gather or simple P2P
   - Single-GPU fallback

6. **Update CMakeLists.txt:** Add distributed directories and sources

7. **Update tests/CMakeLists.txt:** Add test file and include directories

8. **Create `distributed_ops_test.cu`:** Comprehensive tests for all four operations

9. **Build and verify:** Ensure compilation succeeds and tests pass on single-GPU CI

## Success Criteria

1. **All-reduce correct:** Sum of values across GPUs matches expected total
2. **Broadcast correct:** All GPUs receive identical data from root
3. **All-gather correct:** All GPUs receive concatenated data from all GPUs
4. **Barrier no deadlock:** Multiple barrier calls complete without hang
5. **Single-GPU fallback:** All operations work correctly with 1 GPU
6. **Event-based sync:** No use of blocking `cudaMemcpyPeer` or `cudaDeviceSynchronize` in hot paths
7. **Tests pass:** All tests pass on single-GPU CI runner

## Pitfalls Addressed

| Pitfall | Mitigation |
|---------|------------|
| PITFALL-3: Synchronization Deadlocks | Event-based sync with per-device streams; DAG communication pattern |
| PITFALL-2: Memory Consistency | Explicit event synchronization before cross-GPU reads |
| PITFALL-6: Single-GPU Fallback | All operations check device count and take single-GPU path |
| PITFALL-8: Stream Per Device | MeshStreams maintains one stream per GPU |
