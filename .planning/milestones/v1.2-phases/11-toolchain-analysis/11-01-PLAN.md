## Audit Tasks

### Task 1: C++ Standard Compatibility Analysis

**Check:**

1. All C++20 features used in codebase
2. C++23 compatibility of each feature
3. Any deprecated features removed in C++23

**C++23 Key Changes:**

- `std::expected` - error handling
- `std::print` - formatted output
- `std::to_underlying` - enum conversion
- `std::views::chunk_by` - range grouping
- `constexpr` extensions
- `import std` module (optional)

### Task 2: CUDA Standard Compatibility Analysis

**Check:**

1. CUDA 17 features used in codebase
2. CUDA 20 compatibility assessment
3. `__CUDA_ARCH__` usage for compute capabilities

**CUDA 20 Considerations:**

- New CUDA language features
- Potential deprecations in CUDA 17 APIs
- Architecture support (compute capability 10.x?)

### Task 3: CMake Version Compatibility Analysis

**Check:**

1. All CMake 3.25+ specific features used
2. CMake 4.0+ compatibility
3. `GENERATE` property syntax (new in CMake 3.26)

**CMake 4.0+ Breaking Changes to Check:**

- Policy changes for `cmake_minimum_required`
- Generator expression changes
- Module deprecations
